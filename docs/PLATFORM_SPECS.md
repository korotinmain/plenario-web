Design a premium, modern, highly polished web app dashboard for **Plenario** — a personal planning workspace focused on projects, tasks, daily execution, and calm productivity.

## Product direction

Plenario is not just a task manager. It should feel like a **planning-first personal workspace** that helps users understand:

- what matters today
- what is overdue or at risk
- which projects are moving
- what to do next

The design must feel like a real high-end SaaS product, not a template. It should combine:

- calm clarity
- strong information hierarchy
- premium visual polish
- product depth
- elegant whitespace
- subtle emotional warmth

## Style direction

Create a **world-class SaaS UI** with:

- premium modern dashboard aesthetics
- clean grid-based layout
- refined typography hierarchy
- soft shadows
- large rounded corners
- subtle borders
- elegant spacing rhythm
- tasteful iconography
- minimal but distinctive branding
- balanced use of blue as primary brand color
- semantic colors for status:
  - red for overdue / risk
  - amber for warning / upcoming
  - green for done / healthy
  - blue for primary actions and active state

Avoid generic template feel. Avoid clutter. Avoid overly playful startup visuals.
The result should feel like a mix of:

- Linear clarity
- Notion calmness
- modern enterprise SaaS polish
- productivity-focused UX

## Core UX goal

The dashboard should feel like a **daily command center**, not just an analytics page.

## Main layout

Create a left sidebar and a main content area.

### Sidebar

Include:

- Plenario logo and wordmark
- short tagline: “Personal planner”
- primary nav:
  - Dashboard
  - Tasks
  - Projects

- secondary section:
  - My Projects
  - small project items with colored dots

- bottom account area:
  - Settings
  - current user avatar, name, and email

The sidebar should feel premium, clean, soft, and well-spaced.

## Dashboard content

Design a much stronger dashboard structure than a typical CRUD screen.

### Section 1: Hero / Daily Command Center

Large top hero card with:

- greeting: “Good afternoon, Denys”
- current date
- summary of today:
  - overdue items
  - planned tasks
  - active projects

- one strong primary CTA: **Plan today**
- one secondary CTA: **New task**

This section must immediately communicate purpose and momentum.

### Section 2: Status Summary

Compact KPI cards or pills showing:

- Overdue
- Due today
- In progress
- Completed this week

These should be informative but secondary to the hero section.

### Section 3: Priority Now

A powerful working block that shows:

- overdue items
- today’s next tasks
- next recommended action

This section should feel operational and alive, not empty or static.

### Section 4: Projects in Motion

Show active projects as rich cards or rows with:

- project color
- project name
- progress
- open tasks count
- next due item
- updated recently indicator

This must feel more meaningful than a plain list.

### Section 5: This Week

A planning-oriented block with:

- upcoming tasks
- deadlines
- inactive projects needing attention
- weekly progress summary

### Section 6: Quick Capture

A compact elegant quick-add area for creating a task fast:

- task title input
- project selector
- due date selector
- add action

## Project Details Page

Also design a premium project details page for a project called **Work**.

### Project header

Include:

- project icon
- project name
- status chip: Active
- created / updated metadata
- compact progress summary
- edit action

The header should be visually strong but not oversized.

### Health summary row

Show high-signal metrics:

- Progress
- Open
- Overdue
- At risk

### Main content area

Split into two columns:

#### Left column

Main tasks workspace:

- grouped tasks
- overdue / today / upcoming grouping
- clear section headers
- elegant empty or partially filled states

#### Right column

Project context panel:

- recent activity
- next milestone
- notes
- quick add task
- lightweight insights

## Empty states

Design premium, guided empty states that teach the user what to do next.
Do not use bland “No tasks yet” placeholders.
Instead make them feel helpful, intentional, and product-driven.

## Brand feel

Plenario should feel:

- structured
- calm
- focused
- slightly premium
- modern
- trustworthy
- polished enough for App Store / SaaS launch quality

## Visual quality bar

The design should look like a final polished product shot from Behance or Dribbble, but still remain realistic and usable.
It must feel production-ready, premium, and cohesive.

## Output

Generate:

1. a high-fidelity dashboard screen
2. a matching project details screen
3. both in the same visual system

Use realistic content, beautiful typography, subtle depth, elegant color accents, and excellent layout composition.
